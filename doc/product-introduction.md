# AI 自然语言转 SQL 平台

## 产品概述

**AI 自然语言转 SQL 平台** 是一款基于语义理解与 AI 驱动的智能数据分析系统。用户只需用自然语言描述业务问题，系统即可自动完成语义解析、SQL 生成、数据查询与可视化展示，让非技术人员也能轻松获取数据洞察。

---

## 核心价值

| 价值维度 | 说明 |
|---------|------|
| **降低数据门槛** | 无需掌握 SQL，业务人员用自然语言即可查询数据 |
| **提升分析效率** | 从提问到获取结果，全程自动化，秒级响应 |
| **语义理解精准** | 规则引擎 + LLM 双引擎解析，支持复杂业务语义 |
| **架构灵活可扩展** | 分层解耦设计，支持多数据源、多模型接入 |

## 核心功能

### 1. 自然语言转 SQL

用户输入中文问题，系统自动识别意图、提取指标与过滤条件，生成可执行的 SQL 语句。

**示例：**

| 用户输入 | 系统理解 |
|---------|---------|
| "查询总销售额" | `SELECT SUM(amount) FROM demo_orders` |
| "本月采购订单数" | `SELECT COUNT(*) FROM demo_orders WHERE order_type=1 AND create_time BETWEEN ...` |
| "北京地区正常订单" | `SELECT * FROM demo_orders WHERE order_status=1` |

### 2. 双引擎语义解析

| 引擎 | 特点 | 适用场景 |
|------|------|---------|
| **规则引擎** | 快速、稳定、可预测 | 简单查询、高频场景 |
| **LLM 引擎** | 灵活、支持复杂语义 | 多维度查询、模糊表达 |
| **混合引擎** | 先 LLM 后规则回退 | 生产环境最佳实践 |

### 3. AI 智能对话

基于 Chainlit 构建的 AI 对话界面，支持：

- 多轮对话与上下文理解
- 自动图表推荐与生成
- 语义解析过程可视化
- SQL 语句透明展示
- 数据表格与 KPI 卡片

### 4. Cube 语义建模

支持预定义业务语义模型（Cube），将业务概念映射为底层数据结构：

- 指标定义（如 `amount`、`order_cnt`、`profit`）
- 维度定义（如 `order_type`、`order_status`、`create_time`）
- 过滤条件映射
- 缓存加速

### 5. 安全与权限（RBAC + ABAC）

| 权限类型 | 检查时机 | 控制粒度 |
|---------|---------|---------|
| **RBAC** | MCP Hub 入口 | 角色级（谁能做什么操作） |
| **ABAC** | Semantic Layer 之后 | 数据级（谁能看到什么数据） |

- **认证服务**：Token 验证 + 外部认证接口框架
- **RBAC 角色控制**：admin / analyst / viewer
- **ABAC 数据过滤**：根据用户区域属性动态追加 WHERE 条件
- SQL 安全控制与请求限流

## 技术亮点

### 分层解耦架构

```
Web UI / Agent → MCP Hub → Auth(RBAC) → Semantic Layer → Policy(ABAC) → Metrics Engine → Database
```

每一层独立部署、独立扩展，支持：

- 单独升级语义解析能力
- 替换底层数据库
- 接入不同 LLM 模型
- 水平扩展服务实例

### 智能编排

Agent Service 作为智能编排中枢：

- 自动选择最优解析策略
- 调用本地 LLM 生成自然语言答案
- 支持多轮对话上下文管理
- 失败自动降级与回退

### 多 UI 入口

| UI 类型 | 技术栈 | 特点 |
|---------|--------|------|
| **Chainlit** | Python + React | AI 对话式，支持流式输出 |
| **Streamlit** | Python | 传统 Web 界面，适合数据分析师 |

## 典型应用场景

### 场景一：业务人员自助查询

> "这个月销售总额是多少？"

系统自动识别时间范围（本月）、指标（销售额），生成 SQL 并返回结果，同时展示趋势图表。

### 场景二：数据分析师快速探索

> "按产品类型统计采购订单的平均金额"

系统识别维度（产品类型）、指标（平均金额）、过滤条件（采购），生成带 GROUP BY 的 SQL。

### 场景三：管理层数据看板

> "对比本月和上月的利润变化"

系统理解时间对比语义，生成相应的 SQL 查询，并以可视化图表展示对比结果。
